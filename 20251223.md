# 클래스와 Static
## Static
`static` 키워드는 `프로그램이 종료될 때 까지 할당 해제되지 않고` 고정된 영역의 메모리의 공간을 할당할 때 사용한다.

게임 내에서 지속적으로 다뤄져야 하는 누적 점수, 혹은 게임 내의 상황이 변화해도 바뀌어선 안 될 설정치 등과 같은 단순 변수의 개념부터 클래스와 같은 참조 타입까지 선언이 가능

이를 메서드에 붙인다면 프로그램 내에서 간편하게 호출할 수 있는 정적 메서드로도 활용할 수 있어 편리한 기능이지만, 당연히 사용시의 주의를 요한다.

### 메모리 상주
static 키워드를 사용하면 해당 요소는 프로그램 종료시까지 메모리에 계속해서 머무르고, 할당 해제되지 않는다. 그렇기 때문에 필요 이상의 객체를 static으로 선언한다면 불필요한 메모리를 계속해서 사용하게 된다.

```csharp
internal class Tester
{
    private StaticStudy _study1 = new StaticStudy();
    private StaticStudy _study2 = new StaticStudy();
    private StaticStudy _study3 = new StaticStudy();
    public void Run()
    {
        StaticStudy.staticvalue = 1;
        _study1.value = StaticStudy.staticvalue;    
        _study2.value = StaticStudy.staticvalue;
        _study3.value = StaticStudy.staticvalue;

        Console.WriteLine(_study1.value);
        Console.WriteLine(_study2.value);
        Console.WriteLine(_study3.value);
    }
}
```
```csharp
 internal class StaticStudy
 {
     //프로그램 전역에서 사용 가능하다.
     //프로그램 종료시까지 메모리에 상주한다.
     public static int staticvalue = 15; //static 필드

     public int value = 99; //일반 필드
     
     public StaticStudy()
     {
         Console.WriteLine("public 생성자");
         staticvalue = 10;
     }
     static StaticStudy()
     {
         //인스턴스 생성 시 = static 먼저, publi이 다음
         //값, 함수에 먼저 접근 = static만 호출
         //접근을 안 하는 경우 = 호출 안 됨
         //여러 인스턴스 생성 시 = 한번만 호출
         //static으로 지정된 데이터만 제어 가능
         Console.WriteLine("Static 생성자");
     }
 }
```
## 상속
### 상속이란?
객체지향 프로그래밍의 특징 중 하나로, 코드의 중복을 줄이고, `유지보수성을 개선`할 수 있는 기능.

상속을 사용하게 되면, 공통점을 묶어 Base Class 즉, 부모 클래스로 두고, 여기서에 클래스를 파생시켜 중복된 코드를 줄이고, 간결하게 작성할 수 있다.
