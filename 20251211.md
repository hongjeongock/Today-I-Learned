# 배열
`동일한 타입의 여러 데이터 요소`를 하나의 `변수`에 저장하고 관리하는데 사용되는 형식

서로 관련이 있고, 자료형이 같을 경우, 하나의 묶음으로 만들 수 있다. 학생들의 성적이나, 아파트의 세대주 명 등 비슷한 계열의 자료를 묶을때, 배열이라는 기능을 사용하면 편리하다.

- 예를 들어 30발의 총알을 가진 총을 가전할 때, 각 총알이 발사 되었는지 판단하려면 30개의 변수가 필요하게된다. 이처럼 같은 목적과 같은 타입을 가진 변수가 여러개 있을 때 배열로 묶으면 효율적이다.

게임 시스템에서 단축키 1, 2, 3, 4 입력에 따라 아이템을 사용하는 기능을 구현할 때, 각 단축키와 그에 해당하는 사용 내용은 모두 문자열(string) 타입이므로 배열로 묶어 관리할 수 있다.

## 배열 선언
문자열 배열을 선언 하려면 자료형 뒤에 대괄호 두개 ([])를 붙여 표현

string만 쓰면 문자열 하나를 저장하는 자료형 이지만,string[][]처럼 대괄호 두개를 쓰면 문자열을 여러개 담는 문자열 배열이 된다.

### 배열 이름 지정
일반 변수 처럼 이름을 지정한다.(예: int[] hortcuts)
### 공간 크기 지정
배열 자료형은 몇개의 공간을 사용하겠다고 알려줘야한다.
- `new` 키워드를 사용하고 뒤에 `String 4`와 같이 원하는 공간의 개수를 입력한다.
### 문법 해석 
프로그래밍 문법은 한국어 직역보다 영어를 읽는 순서와 동일하게 해석하면 더 자연스럽다.

    String [ ] Shortcuts;는 "자료형은 string, 배열을 선언하라, 이름은 Shortcuts로"가 아닌, 거꾸로 해석하는 것이 이해에 도움이 됨. 

    Shortcuts = new String[4]; 역시 "Shortcuts에 대입해야 할 값"으로 보고, New String 4를 앞에서부터 해석하면 "string 형식이 담길 수 있는 공간 4개만큼을 새로 만들어 달라"는 의미로 쉽게 이해할 수 있다.

## 값 타입(Value Type)과 참조 타입(Refernce Type)
- 두가지 자료형 : C#에서 자료형은 `주소(위치정보)를 담을 수 있는 형식`과 `데이터 그 자체를 담는 형식` 두가지로 나뉜다.

### 값 타입
`int`, `long`, `char`등이 여기에 해당한다.

변수에 <marker>직접 숫자와 같은 값<marker>을 대입하는 방식이다.

값 타입은 실체의 그 값 자체가 저장된다.

(예 : 동전 뽑기에서 캡슐 안에 작은 상품이 직접 들어있는 것과 같다.)

### 참조타입
배열(`int[]`, `string[]`)등이 해당 한다.

`new` 연산자를 통해 공간을 확보하면, 해당위치 정보가 반환되며 이 주소값을 변수에 저장한다.

참조 타입은 실제 값이 담기는 것이 아니라 위치 정보를 저장하는 타입이다.

(예 : 캡슐 안에 문방구 내에서 물건을 수령하세요 라는 쪽지(주소값)가 들어있는것과 같다.)
## 배열의 사용법
### 배열 요소에 값 할당하기
- 선언과 할당의 통합

    배열 선언과 초기 할당은 코드를 한줄로 줄여서 작성 할 수도ㅓ 있다.
- 인덱스의 이해

    배열의 공간에 접근할 때는 배열 이름 옆에 대괄호와 숫자를 기입한다.

    숫자를 셀 때는 1부터 시작하지만, 배열의 첫번째 요소는 반드시 `0부터 시작`한다.

    4개의 공간을 만들었다면 접근 가능한 인덱스는 0, 1, 2, 3이다.
- 인덱스 의미 
`new string[4]` : 문자열을 담을 총 4개의 공간을 빌려달라.

`shortcuts[3]` : 배열명 옆에 붙은 숫자로 (숫자 + 1) 번째 변수 공간에 접근 시켜달라.

### 배열의 요소 꺼내 쓰기
사용자가 입력한 숫자에 따라 출력을 다르게 하는 로직을 구현할 때, 배열을 활용할 수 있다.

사용자가 1을 입력하면 배열의 0번째 요소에 접근하여 해당 내용을 출력한다.

if 문을 사용하여 사용자의 입력값(1~4)에 따라 배열 요소에 접근하면, 1에서 4까지는 정상적으로 작동하는 것을 확인할 수 있다.
### 배열 접근시 주의사항
- 범위 초과 문제: 4개짜리 배열을 만들었는데 다섯 번째 요소(인덱스 4)에 접근하려고 시도하면 문제가 발생함

- 에러 발생: 배열의 범위를 벗어난 인덱스에 접근하면, 실행 결과로 "인덱스가 범위를 벗어났다"는 에러가 발생함. 

- 실제 시스템 영향: 만약 이것이 실제 게임이었다면, 에러 메시지가 뜨거나 프로그램이 꺼지거나(크래시) 발생할 수 있음. 

- 안전 장치의 필요성: 이러한 이유로 배열을 다룰 때는 접근하기 전에 예외 처리와 같은 안전장치를 반드시 해두는 것이 매우 중요하다.
