# 디버거
프로그램을 `멈춰가며` 실행해 보는 도구

코드를 한 줄씩 실제로 실행하면서 내부 상태를 확인 할 수 있다.

    - 어떤 순서로 실행되는지
    - 변수에 지금 어떤 값이 들어있는지
    - 왜 내가 생각한 결과와 다른 결과가 나오는지
를 눈으로 확인하게 도와주는 도구임.

## 왜 쓰는가?

코딩하면서 분명 맞게 쓴 것 같은데 결과가 이상할때, 반복문이 몇 번 도는지 헷갈릴때, 변수 값이 언제 바뀌는지 모를때 등 언제 프로그램이 멈추는지 어떨때 변수 값이 어떤지를 정확하게 보며 감으로 추측하는것이 아닌 실제 실행 상태를 확인 하기 위해 사용한다.

## 브레이크포인트(Breakpoint)
프로그램을 특정 줄에서 멈추게 하는 표시

    코드 줄 왼쪽을 클릭 => 빨간 점

    실행(F5)하면 그 줄에서 프로그램이 정지함

- 왜 중요?

    브레이크포인트가 없으면 프로그램은 시작 부터 끝까지 한 번에 실행됨

    브레이크포인트가 있으면 중간에 멈춤 그 시점의 변수 상태를 확인 가능

## 디버그 실행과 일반 실행의 차이
- Ctrl + F5 : 일반 실행 (결과만 확인)

- F5 : 디버그 실행 (중간에 멈추고 상태 확인 가능)

    디버거 사용 시 F5로 실행해야 한다.

## 한줄씩 실행하는 기능

`F10(step Over)` : 현재 줄 실행하고 다음줄로 이동, 함수가 있어도 함수가 안으로 안 들어감, 한줄씩 보는 용도

`F11(Step Into)` : 함수가 있으면 함수 내부로 들어감, 함수 동작을 자세히 보고 싶을때 사용

`Shift + F11(Step Out)` :  함수 안에 있을때 함수 실행 끝내고 밖으로 나옴

## 변수 값 확인하기
디버거가 멈춘 상태 : 

변수 위에 마우스 올려 현재 값 표시, 지역 변수 창, 조사식으로 i가 지금 몇인지 배열에 어떤 값이 들어있는지 를 바로 확인 가능함.

## 조건문(if) 디버깅
조건문이 실행되지 않을 때, 코드가 틀렸다고 단정하면 안된다. `디버거를 통해 조건식에 사용된 변수의 실제 값`, `조건식의 판단 결과`를 확인해야한다. 조건이 맞을 것 같다는 생각은 오류의 원인이 된다.

## 반복문(for / while) 디버깅
반복문에서는 `시작값`, `종료조건`, `증감식`, `실제 반복 횟수`를 확인한다. 반복문 내부에 브레이크 포인트를 찍고 F10으로 실행하면 반복 흐름과 변수 변화가 명확히 보인다.

## 함수 실행 흐름 이해
함수는 동시에 실행 되지 않고 호출 => 실행 => 종료 => 복귀 순서로 동작한다.

디버거를 통해 함수 호출 흐름을 직접 확인 할 수 있다.

함수 안에서 선언된 지역 변수는 해당 함수 실행 중에만 존재하고 함수 종료시 사라짐.

Local Variables 창을 통해 이를 확인할 수 있다.
## ReadLine쪽 디버거 찍는법은?
콘솔의 입력을 받는 ReadLine과 같은 키워드가 나오면 콘솔로 자동으로 포커스가 옮겨간다. 여기서 입력을 정상적으로 진행해주신 후 엔터를 친 후, Visual Studio로 돌아가서 디버깅을 이어가면 된다.
