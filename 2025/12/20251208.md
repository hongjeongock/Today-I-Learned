## 컴퓨터의 기본 구성
컴퓨터를 구동 시키기 위해서는 `소프트웨어`와 `하드웨어가` 있음. 하드웨어는 물리적으로 존재하는 부품(육체) 소프트웨어는 이 하드웨어를 작동 시키는 명령어들로 이루어져있음(영혼)


### CPU
    컴퓨터의 뇌.
    명령어를 해석하고 주로 계산을 수행하는 담당.

### RAM
    cpu와 가장많이 소통.
    휘발성 메모리 - 전원이 꺼지면 저장된 내용이 사라지는 메모리

### 메인보드
    마더보드 라고도 불림 CPU,RAM, 그래픽카드 등의 하드웨어 장치를 연결하는데 사용.

### 그래픽카드
    그래픽 처리 담당
    GPU를 포함하고 있으며, 이미지·영상·3D 그래픽을 처리하는 장치

### 보조기억장치
    프로그램들을 저장하는 역할

### 그 외 입출력 장치
    마우스, 키보드 등 컴퓨터와 사용자간의 상호작용을 위한 다양한 하드웨어 장치

### 소프트웨어
    하드웨어를 제어하고 기능을 수행하는 데 사용되는 지침 및 데이터의 집합
    소프트웨어에는 운영 체제, 응용 프로그램 및 드라이버가 포함

- 응용 프로그램

    일반 사용자들이 사용하는 프로그램으로, 특정한 작업을 수행하기 위해 존재 (.exe파일 등)

- 시스템 프로그램

    응용 프로그램이 돌아가도록 도와주는 관리자 역할

### 컴퓨터가 이해하는 언어

    이진법으로 이루어져있음. 0과1로 모든 데이터를 표현함.
    전기가 흐르면1 전기가 흐르지않으면 0

### 컴퓨터가 동작하는 원리

- 소스코드

- 컴파일(compile)
    인간이 작성한 소스코드를 기계어(0과1)로 변환 해주는 코드

- 오브젝트 코드(Object Code)
    소스코드가 컴파일된 결과물인 오브젝트 코드(obj)는, 기계어로 작성된 이진 형태의 코드 이지만 아직 실행 가능한 형태는 아님. 실행을 위해 추가적인 처리가 필요 링커를 통해 실행파일로 결합되어야 실행됨

- 링커(Linker)
    여러 개의 오브젝트 코드나 라이브러리 들을 하나의 실행 가능한 형태로 묶어주는 프로그램.`

- 실행파일(exe)
    링커를 통해 실행에 필요한 오브젝트 코드와 리소스가 포함된 파일

- 로더(Loader)
    실행을 했을때 메모리(RAM)에 올려주는 전단계.
    로더에 의해 메모리에 올라간 단 하나의 단위는 '프로세스'라고 표현
    운영체제의 관리 하에 실행되는 독립적인 작업 단위로,프로그램이 실행 중인 상태를 나타냄


### 10진법 2진법

전기가 통하면 1, 아니면 0

#### 10진수를 2진수로 바꾸는 방법

숫자 3이 있다고 가정하고 3을 2로 나누면 몫은 1, 나머지 1

#### 2진수를 10진수로 바꾸는 방법

1111로 가정하고 1의 자리부터 10 단위로 올라가기에 자릿수 별로 일치하는 10의 제곱 수 만큼을 곱하여 모두 더하면 값이 나옴

(2³ * 1) + (2² * 1) + (2¹ * 1)+ (2⁰ * 1)

= 8 + 4 + 2 + 1

= 15

### bit와 byte

bit:정보를 저장하고 처리하는데 사용되는 가장 작은 단위

byte : bit를 8개 붙이면 byte가 된다.
