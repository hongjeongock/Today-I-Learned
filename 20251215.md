# 열거형
숫자에 별명(이름표)을 붙여 알아보기 쉽게 만들어 주는 기능

## 왜 필요한가?
컴퓨터는 숫자를 좋아하지만 사람은 의미 있는 말을 좋아한다.

예를들어
`0` => 이게 뭐지?

`Monster` => 아, 몬스터구나~!

때문에 숫자 대신 이름으로 쓰기 위해서 열거형을 만든다.

```csharp
 static void Main(string[] args)
 {
     Console.WriteLine((GameObjectType)0);
 }
 enum GameObjectType
 {
     Monster, Item, NPC, fieldObject
     // 요소가 늘어날 때마다 할당되는 숫자도 늘어나고,
     // 선언한 대로 정수에 별명이 붙는다.
 }
 static void OnCollision(GameObjectType type)
 {
     switch (type)
     {
         case GameObjectType.Monster:
             Console.WriteLine("몬스터와 부딪혔다. 피격처리");
             break;
         case GameObjectType.Item:
             Console.WriteLine("아이템과 부딪혔다. 먹기");
             break;
         case GameObjectType.NPC:
             Console.WriteLine("NPC와 부딪혔다. 말을 건다");
             break;
         default:
             Console.WriteLine("아무 일도 없다");
             break;
     }
 }
```
```csharp
Console.WriteLine((GameObjectType)0);
```
0이라는 `숫자`를 `GameObjectType`이라는 별명(이름표)로 바꿔서 출력해라

## 왜 Switch랑 같이 쓰는가?
"누구랑 부딪혔니?" 라고 묻는거와 같다.

몬스터면 공격, 아이템이면 줍기, NPC면 대화. 숫자 0, 1, 2로 하면 헷갈리지만 Monster, Item, NPC면 사람이 이해 할 수 있다.


# 구조체
`값 형식의 데이터`로 .NET의 기본 자료형은 모두 구조체로 정의되고 있다.

구조체는 관련 있는 값들을 한 덩어리(상자)로 묶어 놓은 것

```csharp
struct Item
{
    public string Name;
    public int Id;
}
```
아이템이라는 상자 안에는 이름(Name)과 번호(Id)가 들어 있다.
```csharp
Item item;
item.Name = "돌";
item.Id = 101;
```
"돌" 이라는 이름과, "101번"이라는 번호를 하나의 묶음으로 관리

## 왜 그냥 변수 여러 개 안 쓰고 구조체를 쓰는지
- 변수만 쓰면
```csharp
string itemName;
int itemId;
```
이게 누구의 이름인지, 번호인지 몇개 없을때는 헷갈리지 않지만 정보가 늘어나면 관리가 힘들어지기 때문이다.

## 값 형식
구조체는 복사하면 그대로 복사 됨.

장난감 하나를 복사하면 완전히 같은 장난감이 생기는 느낌

## private 
`private` : 구조체 내부에서반 접근이 가능하다.
```csharp
private int id;
```
"이 상자 안에 있는건 밖에서 함부로 못만진다."로 이해하면 된다.
