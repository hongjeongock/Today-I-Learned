# 버전 관리
버전관리 도구는 대표적으로 Git을 사용한다.

깃은 기존 작성된 내용에서부터 생긴 변동사항을 추적하고, 변동사항들을 마치 버전처럼 관리할 수 있는 기능을 제공한다.

만약 지금 작업중인 사항에 문제가 있을 경우, 변경사항을 취소하거나 되돌리는 기능을 제공하며, 더 나아가선 `과거 특정 버전의 파일로 되돌아가는` 기능을 제공한다.

깃은 웹상의 원격 저장소에 저장이 되기에, 필요시 다른 기기에서 편리하게 업데이트 된 최신 작업본을 내려받을 수 있다.

유니티와 같은 큰 프로젝트를 진행할 때는 협업을 위해 깃을 자주 사용하게 될 것이다.

## git clone(원격 저장소 복제)

GitHub Repository 생성과 GitHub 인증 설정을 완료했다면, 이제 GitHub에 있는 저장소를 로컬로
가져올 차례다. git clone은 원격 저장소를 통째로 복제해서 로컬에 가져오는 명령어다.

clone을 하면 원격 저장소와 자동으로 연결된 상태로 시작할 수 있어서, 별도로 원격 저장소를 추가하는 작
업 없이 바로 작업을 시작할 수 있다.

## git clone이 하는 일

git clone을 실행하면 다음 작업들이 자동으로 수행된다.

1. 원격 저장소의 모든 파일과 커밋 이력을 로컬로 복사
2. 원격 저장소와의 연결 정보 자동 설정 (origin)
3. main 브랜치로 자동 체크아웃

• 현 단계에서는 main 작업공간으로 이동한다고 생각하면 된다.

즉, clone 한 번이면 모든 준비가 끝난다.

## git commit (변경사항 기록)

git add (스테이징)로 스테이징 영역에 파일을 올렸다면, 이제 변경사항을 기록할 차례다. `git commit`은 스테이징된
변경사항을 로컬 저장소에 영구적으로 기록하는 명령어다.

커밋은 00. Git에서 가장 핵심적인 개념이다. 프로젝트의 특정시점을 스냅샷으로 저장하고, 나중에 언제든지 그 시점으로 돌아갈 수 있게 해준다.

## git push (원격 저장소에 업로드)

git commit (변경사항 기록)으로 로컬 저장소에 변경사항을 기록했다면, GitHub Repository 생성 & git
clone (원격 저장소 복제) 혹은 git remote (원격 저장소연결) 과정을 통해 연결한 원격 저장소에 올릴 차례다. git push는 로컬의 커밋들을 원격 저장소로 전송하는 명령어다.

Git으로 협업하는 상황이라면 다른 사람들이 내 작업 내용을 볼 수 있게 하고, 혼자 작업한다 해도 commit한 내용을 반드시 push 해야 한다.
push 전에는 로컬에 있는 작업 이력일 뿐이다.

## 커밋메세지 규칙
커밋 메세지는 보고서에 작성하는 `제목` 이라고 생각하면 된다

## git checkout (커밋 이력으로 되돌리기)

git commit (변경사항 기록)으로 쌓아온 커밋 이력을 따라 과거로 돌아가고 싶을 때가 있다. "저 버전에선 잘 됐는데 ... "싶을 때, 혹은 특정 시점의 코드를 확인하고 싶을 때 git checkout을 사용한다.

이는 00. Git을 활용한 버전관리에서 지원하는 상당히 강력하고 주된 기능으로, git checkout은 원래 브랜치를 전환하는용도로 많이 쓰였지만, 이 글에서는 커밋 이력으로 되돌리는 기능에 집중해서 다룬다.               
## git fetch (원격 변경사항 확인)

git remote (원격 저장소 연결)로 원격 저장소와 연결했다면, 이제 원격 저장소의 변경사항을 확인할 차례다. 
git fetch는 원격 저장소의 최신 정보를 로컬로 가져오는 명령어
다.

협업을 하다 보면 다른 사람이 코드를 수정하고 원격 저장소에 올렸는데, 내 로컬에는 아직 그 변경사항이 반영되지 않은 상태가 된다. 이때 git fetch를 사용하면 원격의 변경사항을 확인할 수 있다.

## .gitignore (추적 제외)
00. Git으로 프로젝트를 관리하다 보면 Git에 올리면 안 되는 파일들이 있다. 빌드 결과물,
임시 파일, 개인 설정, 민감한 정보가 담긴 파일 등이 대표적이다. 이런 파일들을 실수로
git commit (변경사항 기록)하면 저장소가 지저분해지거나, 심각한 경우 보안 문제가
생길 수도 있다.

·gitignore 파일은 이렇게 Git이 추적하지 말아야 할 파일들을 지정하는 설정 파일이다.
·gitignore에 등록된 패턴과 일치하는 파일은 07. git status (상태 확인)에 나타나지 않
고, 08. git add (스테이징)로 추가할 수 없다.
