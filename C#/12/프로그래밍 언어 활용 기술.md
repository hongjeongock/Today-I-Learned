### 5. 비교 연산자( = 관계형 연산자)

### 6. 논리 연산자

두개 이상의 bool타입 데이터에 대한 논리 연산을 수행한다.

크게는 사용 목적에 따라 Bool 논리연산자, 조건부 논리연산자로 나눠볼 수 있다.

다만, 조건부 논리연산자 쪽이 효율적으로 형태로 동작하기 때문에 자주 쓰이는 방식이다.

그럼에도 불구하고 bool 논리연산자는 이진수 타입을 연산해야 할 때도 사용되므로 알고 있어야 할 개념이다.

연산자는 기본적으로 이진 형태의 데이터, bool 타입에 대해 수행하는 것이 일반적이다.

| 연산자  |    예     | 의미  |                     설명                      |
| :--: | :------: | :-: | :-----------------------------------------: |
|  &&  |  A && B  | AND | A항과 B항 **모두** True일 때만 True, 그렇지 않으면 False  |
| \|\| | A \|\| B | OR  | A항과 B항 **모두** False일 때만 False, 그렇지 않으면 True |
|  !   |    !A    | NOT |    A항이 True이면 False, A항이 False면 True를 반환    |
- 점프
	1. 키 눌림
	2. 바닥에 붙어있어야 함
	3. 캐릭터 살아있음
	4. 다른 상태가 아닌 경우

```csharp
bool isKeyDown = true;
bool isGrounded = true;
bool isAlive = true;
bool isAnotherState = false;

bool canJump = isKeyDown && isGrounded && isAlive && ! isAnotherState;
```

>비트연산자와의 차이점은, Bool 논리연산자는 실제로 주어진 모든 데이터에 대한 논리연산을 수행하지만, 조건부 논리 연산은 이후의 논리연산을 수행할 필요가 없다면 수행하지 않는다.

### 7. 조건부 논리 연산자(비트 연산자)

| 연산자 |      의미      |             설명              |
| :-: | :----------: | :-------------------------: |
|  &  |   논리곱(AND)   | 비트 값이 둘 다 1일 때만1, 그렇지 않으면 0 |
| \|  |   논리합(OR)    | 비트 값이 하나라도 1이면 1, 그렇지 않으면 0 |
|  ^  | 배타적 논리합(XOR) | 비트 값이 서로 다를 때만 1, 그렇지 않으면 0 |
|  ~  |     NOT      |      비트 값이 1이면0, 0이면 1      |

```csharp
bool a = true;
bool b = false;
bool c = true;
bool d = false;

bool result = a | b | c | d;

Console.WriteLine(result);
```

```csharp
bool a = true;
bool b = false;
bool c = true;
bool d = false;

bool result = a & b & c & d;

Console.WriteLine(result);
```

### 8. 연산자 우선순위


| 항목     |         연산자          | 우선순위 |
| ------ | :------------------: | :--: |

| 증감 연산자 |        ++, --        |  \|  |
| 산술 연산자 |          -           |  \|  |
|        |         *, /         |  \|  |
|        |          %           |  \|  |
|        |         +, -         |  \|  |

| 관계 연산자 | ==, !=, <, >, <=, >= |  \|  |

|        |       \|\|(OR)       |  ↓   |
